# PRD — RF4 어종 통계 사이트

문서 버전: v1.1 (2026-06-12)
상태: 검수 대기
변경: v1.1에서 제품 포지셔닝 피벗 — 전체 기록 브라우저 제거, 선호 어종 대시보드 중심으로 재정의 (D-10)

---

## 1. 배경과 문제 정의

러시안피싱4는 어종별 활성 포인트와 선호 미끼가 계속 바뀌지만, 이 정보를 공식적으로 제공하지 않는다. 유저는 커뮤니티 입소문이나 직접 탐색에 의존해야 하며, 공식 사이트의 주간기록은 단편적인 스냅샷일 뿐 추세를 보여주지 않는다.

단순 전체 기록 조회만으로는 차별점이 없다. **이 제품의 차별점은 개인화다: 내가 노리는 선호 어종들만 등록해두면, 접속 순간 그 어종들의 활성화 상태가 자동으로 정리되어 "오늘 뭘 잡으러 갈지"를 즉시 결정하게 해주는 개인 관제 대시보드.**

## 2. 사용자

- 1차: 제작자 본인 (로컬 개인 사용)
- 2차(추후): 한국 RF4 유저 커뮤니티 (공개 배포 시)

사용자의 핵심 순간: **게임 접속 직후, "오늘 뭐 잡으러 가지?"를 5분 안에 결정하고 싶은 순간.**

## 3. 유저 스토리

우선순위: P0 = 1차 출시에 필수 / P1 = 중요하나 후순위 가능 / P2 = 보류

| ID | 우선순위 | 스토리 |
|----|---------|--------|
| US-4 | **P0** | 선호 어종을 등록해두고 그 어종들만 한 화면에서 보고 싶다. 이 제품을 쓰는 이유 자체. |
| US-6 | **P0** | 접속하자마자 **내 선호 어종 중** 오늘 잡을만한 것을 추천받고 싶다. 어종마다 활성화 강도가 다르므로. |
| US-5 | **P0** | 추천받은 어종이 지금 어떤 미끼·장소에서 나오는지 보고 싶다. 추천만으로는 출조 준비가 안 되기 때문에. |
| US-3 | P1 | 내 어종들의 트로피급 기록 흐름을 따로 보고 싶다. 일반 크기는 관심 없기 때문에. |
| US-2 | P1 | 대시보드를 수역 기준으로 좁혀 보고 싶다. 오늘 갈 맵이 이미 정해진 날도 있기 때문에. |
| ~~US-1~~ | 제거 | ~~전체 기록 브라우저~~ — 단순 기록 조회는 차별점이 아님 (D-10) |

### 수용 기준

**US-4 선호 어종 (제품의 뼈대)**
- 전체 어종 목록(수집된 어종)에서 검색해서 별표로 등록/해제할 수 있다
- 메인 화면은 로그인한 회원의 선호 어종만으로 구성된다 (회원별 분리, D-15)
- 선호 어종이 0개인 첫 방문 시: 어종 선택을 안내하는 온보딩 화면이 보인다
- 수집 전이라 어종 목록이 비면 "수집 대기 중" 안내가 보인다

**US-6 오늘의 타겟 (메인 화면)**
- 선호 어종들이 활성도 점수 순으로 카드 정렬된다 (강한 활성이 맨 위)
- 각 카드에 추천 근거가 보인다: 활성 상태, 트로피 건수, 미끼 일관성 %, 대표 장소, 대표 미끼
- 비활성 어종도 숨기지 않고 하단에 가라앉혀 보여준다 ("얘는 오늘 아니다"도 정보다)

**US-5 어종 상세 (출조 준비)**
- 카드 클릭 시 해당 어종의 선택된 시간창(오늘=24h/3일=72h) 미끼별 기록 수 순위가 보인다 (색상코드 구분)
- 장소별 분포가 함께 보인다
- 트로피이상 기록만 보기 토글이 있다

**US-3 트로피 흐름 (P1)**
- 내 어종들의 트로피급 이상 기록이 시간순으로 보인다
- "레어트로피만" 토글이 있다

**US-2 수역 필터 (P1)**
- 대시보드를 특정 수역 기준으로 좁힐 수 있다 (그 맵에서의 내 어종 활성도만)

## 4. 용어집 (수치 정의)

| 용어 | 정의 |
|------|------|
| 기록 (catch) | 주간기록에서 수집된 물고기 1마리. 중복 키 (player, species, weight_g, caught_date) |
| 등급 | 일반 / 트로피 / 레어트로피. 어종별 기준 무게(trophies 테이블) 이상 여부로 판정 |
| 트로피이상 | 등급이 트로피 또는 레어트로피인 기록 |
| 모집단 | 시간창 내 그 어종의 **전체 수집 기록** (무게 하한 없음). 작은 기록이 갱신 안 되고 남아있는 것 자체가 "큰 게 안 나온다=비활성"의 근거이므로 버리지 않음 (D-21) |
| 미끼 일관성 | 해당 어종의 **전체 기록** 중 최다 미끼의 점유율 (0~1) |
| 무게 비율 (ratio) | 기록의 무게 ÷ 트로피(또는 레어) 기준 무게. 등급 3단계로 자르지 않고 연속값으로 보존. 라벨 스냅샷에 최대/최소/평균 저장 (D-20) |
| 리셋 경과시간 | 직전 주간 리셋(월 04:00 KST)부터 경과 시간(0~168). 주 초반/주말의 활성 추세 차이를 담는 학습 피처 (D-23) |
| 활성 상태 | **강한 활성**(트로피이상 다수 + 일관성↑) / **활성**(기록은 통일됐으나 트로피 적음) / **불명**(일관성↓, 미끼 섞임) / **비활성**(기록이 최소 표본 미만). ※ 임시 수식이며 향후 라벨 기반 ML로 교체 (D-21) |
| 최소 표본 | 추천 대상이 되기 위한 시간창 내 전체 기록 최소 건수. **잠정 5건** — 조정 가능 파라미터 |
| 미끼 단위 | bait_icon title 원본 문자열 전체. 루어 색상코드까지 포함 (예: Veikko 25g-004). **색상이 다르면 다른 미끼** — 인게임에서 전혀 다르게 작용함 (결정 D-7) |
| 기본 시간창 | first_seen(수집 시각) 기준 롤링. 오늘=최근 24시간, 3d=최근 72시간 (D-17). 자정 경계 없음 |
| 활성도 점수 | (임시) 등급 가중합 × 미끼 일관성. 향후 라벨 기반 ML로 교체 (D-21) |

## 5. 기능 범위

### 1차 출시 (P0)
- 선호 어종 등록/관리 + 온보딩 (US-4)
- 오늘의 타겟 대시보드 (US-6)
- 어종 상세 — 미끼/장소 (US-5)

### 2차 (P1)
- 트로피 흐름 보기 (US-3)
- 수역 필터 (US-2)

### 보류 (P2)
- 전체 기록 브라우저 (제거됨 — 필요해지면 P2로 부활 검토)
- 미끼 상세 페이지, 훅베이트/향 분리 집계, 미끼 카테고리 매핑
- 어종별 시계열 추이 (메타 변화 감지)
- 유저 제보 입력 (좌표 수집 여부 미정)
- 인기 어종 보정 통계

## 6. 데이터

### 수집 (구현 완료)
- 출처: rf4game.kr 주간기록 (서버 렌더링 HTML)
- 대상: 3카테고리(records/ultralight/telestick) × 10지역 = 30게시판
  - 라이트/바텀라이트/해양은 별도 주간 테이블이 없음(통합과 동일)이 확인되어 제외
- 전 지역 수집 이유: 주 후반 탑5 진입 장벽 상승에 따른 표본 감소를 지역 수로 완화
- 주기: 수동(현재) → 서비스 시 15분 (APScheduler)
- 예의: 요청 간 1.5초, 403/429 시 사이클 중단, 연속 5회 실패 시 중단

### 스키마 (구현 완료)
```
catches      species, weight_g, waterbody, bait, player, caught_date,
             source('weekly_record'|'user_report'), first_seen
             (first_seen 7일 이내만 유지, 초과분은 archive.db로 — D-19)
appearances  catch_id, category, region, rank, seen_at
trophies     species, trophy_g, rare_trophy_g
users        id, username, password_hash, created_at  (D-15)
favorites    user_id, species  (회원별 분리, D-15)
labels       user_id, species, label, window + 학습 피처 스냅샷
             (등급 개수, 일관성, ratio 통계, 리셋 경과시간 — D-18,20,23)
             영구 보존, 7일 정리 대상 아님
```

## 7. 기술

- Python + FastAPI / SQLite / Jinja2 + 최소 JS / Chart.js(필요 시)

- 코랩 사용 시 rf4.db는 구글 드라이브에 보관
