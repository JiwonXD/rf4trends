# RF4 어종 통계 사이트 기획서

최종 수정: 2026-06-12

## 1. 목표

러시안피싱4의 주간기록 데이터를 수집·누적해서, **"지금 어디서 어떤 미끼로 무슨 물고기가 잘 나오는지"** 를 통계로 보여주고 추천하는 사이트.
공식적으로 제공되지 않는 활성 포인트/선호 미끼 정보를 데이터로 복원하는 것이 핵심 가치.

운영 형태: 당분간 로컬 개인용 → 추후 공개 웹사이트 배포 (코드는 처음부터 배포 가능한 구조로 작성).

## 2. 데이터

### 2.1 수집 (구현 완료)

- 출처: rf4game.kr 주간기록 (서버 렌더링 HTML 파싱)
- 대상: 3개 카테고리(통합 `records` / 울트라라이트 `ultralight` / 텔레스코픽 `telestick`) × 10개 지역 = 30개 게시판
  - 라이트/바텀라이트/해양은 사이트가 별도 주간 테이블을 제공하지 않음(통합과 동일 데이터)이 확인되어 제외 (2026-06 확인)
- 모든 지역을 수집하는 이유: 주 후반으로 갈수록 탑5 진입 장벽이 높아져 표본이 줄어드는 편향을 지역 수로 완화
- 주기: 수동 실행(현재) → 서비스 시 15분 간격 (APScheduler)
- 예의: 요청 간 1.5초 딜레이, 403/429 수신 시 사이클 즉시 중단, 연속 5회 실패 시 중단

### 2.2 스키마 (구현 완료)

```
catches      물고기 한 마리 = 한 행 (중복 제거된 원본)
             species, weight_g, waterbody, bait, player, caught_date,
             source ('weekly_record' | 'user_report'), first_seen
             중복 키: (player, species, weight_g, caught_date)

appearances  그 기록이 노출된 게시판 이력
             catch_id, category, region, rank, seen_at

trophies     어종별 트로피 기준 (출처: rf4kr.com, 정적 데이터 1회 추출)
             species, trophy_g, rare_trophy_g
             등급: 일반 / 트로피 / 레어트로피

favorites    선호 어종 목록 (예정)
             species
```

### 2.3 미끼 데이터 정책 (결정됨)

- **루어는 색상코드까지 구분** 해서 집계 (예: `Veikko 25g-004`) — 색상별 반응 차이가 게임 메타상 유의미
- 보일리+밑밥 조합(`크랜베리 팝업 26; 크랜베리 레드`)은 원본 문자열 그대로가 기본 단위
- 세미콜론 분리(훅베이트/향 별도 집계)는 2차 분석 기능으로 보류
- 출처 사이트 통일: rf4game.kr 한 곳만 사용 (이름 표기 일관성)

## 3. 화면 / 기능

### 3.1 메인 — 전체 기록 브라우저 (1차 범위)

- 처음 화면에서 **모든 기록**이 보임 (최신순)
- 필터: **수역(맵) / 지역 / 어종** 조합 선택으로 좁혀보기
- 각 기록 표시: 어종, 무게, 트로피 등급 뱃지(일반/트로피/레어), 장소, 미끼, 날짜
- 정렬: 최신순 / 무게순

### 3.2 선호 어종 (1차 범위)

- 어종을 즐겨찾기로 등록/해제
- "내 어종만 보기" 필터
- 선호 어종 기준 추천 모아보기
- 로컬 단독 사용이므로 로그인 없이 DB 저장 (공개 전환 시 브라우저 저장/계정으로 확장)

### 3.3 추천 (1차 범위)

두 가지 축, 기본 시간창 **최근 3일**:

| 축 | 질문 | 로직 |
|---|---|---|
| 활발한 조합 | 지금 뭐가 잘 나오나 | 어종×장소×미끼 기록 수 상위 |
| 트로피 조합 | 지금 어디서 대물이 나오나 | 트로피급 이상 기록 수 상위 (레어만 보기 옵션) |

- 시간창은 3일/7일 전환 가능
- 주간 리셋(월요일) 경계 처리: **리셋 무시, caught_date 기준 롤링 윈도우** (결정됨)
  - DB는 게시판 리셋과 무관하게 전 기록을 영구 누적하므로, 월요일의 3일 창에는 토·일 기록이 자연스럽게 포함됨. 별도 처리 불요
  - 알려진 한계: 주말은 탑5 진입 장벽이 높아 표본이 얇음 (원천 데이터의 성질로 수용)

### 3.4 2차 이후 보류 기능

- 미끼 상세 페이지 ("이 미끼로 뭐가 잡히나")
- 훅베이트/향 분리 집계, 미끼 카테고리 매핑 (rf4kr.com 미끼 목록 활용)
- 어종별 시계열 추이 (메타 변화 감지)
- 유저 제보 입력 (source='user_report') — **좌표 수집 여부 미정**
- 인기 어종 보정 (어종 내 비중 % 통계)

## 4. 기술

- 백엔드: Python + FastAPI
- DB: SQLite (`rf4.db`) → 트래픽 증가 시 PostgreSQL 이전
- 프론트: Jinja2 템플릿 + 최소한의 JS (필터), 차트 필요 시 Chart.js
- 수집기: 현재 수동 실행 (코랩/로컬), 서비스 시 APScheduler 15분 주기
- 배포(추후): Railway / Fly.io 등 저가 호스팅, 수집기는 웹서버와 동일 프로세스에서 구동
- 코랩 사용 시 주의: rf4.db는 구글 드라이브에 보관 (런타임 초기화 시 유실 방지)

## 6. 원칙

- 모든 기능은 catches 테이블 하나를 기준으로 파생 (제보 데이터도 같은 테이블, source로 구분)
- 수집 출처는 rf4game.kr로 통일, 정적 게임 데이터는 1회 추출 방식
- 외부 사이트(rf4kr.com 등) 데이터 사용 시 공개 전환 전에 출처 표기 및 양해
- 단순하게: 지금 필요 없는 추상화/기능은 만들지 않는다
